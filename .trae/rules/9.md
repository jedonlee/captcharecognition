# API 服务规范

## 接口设计原则
- RESTful 风格，资源导向
- 统一响应格式：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {...},
    "timestamp": 1234567890
  }
  ```
- 错误响应包含错误码和详细信息

## 本项目 API 端点
- `GET /health` — 健康检查
- `POST /predict` — 单图识别
- `POST /predict/batch` — 批量识别

## 输入校验
- 文件类型校验：只接受图片格式（PNG, JPG, JPEG, BMP）
- 文件大小限制：最大 10MB
- 批量请求限制：最多 100 张图片
- 图片尺寸校验：宽度和高度在合理范围内

## 响应时间要求
- 单图识别：< 100ms（GPU）/ < 500ms（CPU）
- 批量识别：< 50ms/张（GPU）/ < 200ms/张（CPU）
- 健康检查：< 10ms

## 错误处理
- 400 Bad Request：输入参数错误
- 413 Payload Too Large：文件过大
- 415 Unsupported Media Type：不支持的文件类型
- 500 Internal Server Error：服务器内部错误
- 503 Service Unavailable：服务不可用（模型未加载）

## 限流策略
- 单 IP 每分钟最多 100 次请求
- 批量接口每分钟最多 10 次请求
- 超出限制返回 429 Too Many Requests

## 日志记录
- 记录每个请求的：请求ID、IP、端点、耗时、状态码
- 错误请求记录详细错误信息
- 慢请求（> 1s）记录警告

## 监控指标
- 请求量（QPS）
- 响应时间（P50, P95, P99）
- 错误率
- 模型推理时间
- GPU 利用率

## 版本管理
- API 版本通过 URL 路径管理：`/v1/predict`
- 新版本发布时旧版本保持兼容至少 3 个月
- 废弃接口返回 Deprecation Header
