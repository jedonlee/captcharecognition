# 测试规范

## 测试层次
- 单元测试：测试单个函数/类的功能
- 集成测试：测试模块间的协作
- 端到端测试：测试完整流程（从输入到输出）

## 测试覆盖要求
- 核心模块（models/, utils/）覆盖率 ≥ 80%
- API 接口测试覆盖率 100%
- 关键路径（训练、评估、推理）必须有测试

## 测试命名规范
```python
def test_<功能>_<场景>_<预期结果>():
    pass

# 示例
def test_beam_search_decode_valid_input_returns_correct_text():
    pass
```

## 测试数据管理
- 测试数据与生产数据分离
- 使用 fixtures 管理测试数据
- 测试后清理临时文件

## 本项目关键测试点
- 模型前向传播：输入形状、输出形状、log_softmax 值域
- CTC 解码：greedy、beam search、后处理纠错
- 配置加载：必填项缺失、类型错误、路径不存在
- 数据集：图像加载、标签解析、数据增强
- API：正常请求、异常输入、并发请求

## 测试执行
- 修改代码后必须运行相关测试
- 提交前运行全量测试
- 测试失败必须修复后才能提交

## Mock 策略
- GPU 相关测试使用 CPU 模式
- 文件系统测试使用临时目录
- 网络请求测试使用 mock

## 测试报告
- 记录测试覆盖率
- 记录失败用例及原因
- 定期审查测试用例有效性
