# 错误处理与异常规范

## 异常层次
```python
class CaptchaError(Exception):
    """项目基础异常"""
    pass

class ConfigError(CaptchaError):
    """配置相关异常"""
    pass

class DataError(CaptchaError):
    """数据相关异常"""
    pass

class ModelError(CaptchaError):
    """模型相关异常"""
    pass

class TrainingError(CaptchaError):
    """训练相关异常"""
    pass
```

## 异常处理原则
- 禁止裸 except：必须指定异常类型
- 禁止静默吞掉异常：必须记录日志或重新抛出
- 关键操作必须有异常处理：
  - 文件读写
  - 网络请求
  - GPU 操作
  - 模型加载/保存

## 重试机制
- 网络请求：最多重试 3 次，指数退避
- 文件读写：最多重试 2 次，间隔 1 秒
- GPU 操作：OOM 后清理显存并重试 1 次

## 降级策略
- 预训练权重加载失败 → 使用随机初始化，记录警告
- 配置文件读取失败 → 使用默认值，记录警告
- TensorBoard 写入失败 → 跳过 TensorBoard，不影响训练

## 错误信息规范
- 错误信息必须包含：
  - 错误发生的位置（文件名、函数名）
  - 错误原因
  - 建议的解决方案
- 示例：`ConfigError: config.yaml 缺少必填项 'training.batch_size'，请检查配置文件`

## 资源清理
- 使用 try-finally 或 context manager 确保资源释放
- 文件操作后必须关闭
- GPU 异常后必须清理显存
- 训练中断后必须保存 checkpoint

## 禁止
- 使用 sys.exit() 退出（除 __main__ 入口外）
- 在库代码中使用 input() 等待用户输入
- 忽略异常而不记录日志
