# 进程管理规则

## 启动前检查
- 启动新训练/服务前，必须先确认无旧进程在运行
- 检查命令：`ps aux | grep python | grep -v grep`

## command_id 管理
- 每次 RunCommand 启动训练/服务后，必须记录返回的 command_id
- 注意：command_id 仅在当前 session 内有效，对话中断/刷新后失效
- command_id 失效时的备用方案：`pkill -9 -f "python.*train"`

## 停止命令优先级（从高到低）
1. StopCommand(command_id="...")  ← 首选
2. pkill -9 -f "python.*train"   ← 备用
3. kill -9 $(ps aux | grep python | grep train | grep -v grep | awk '{print $2}')

## 双重验证
- 停止命令后，必须执行：`ps aux | grep python | grep -v grep`
- 确认输出为空才算真正停止

## 训练完成后
- 训练正常结束后，检查是否有残留进程
- 如有残留，立即清理

## 训练中断处理
- 训练异常中断后，先保存当前 checkpoint
- 记录中断原因（OOM、手动终止、系统错误等）
- 恢复训练时从 latest_checkpoint.pth 加载

## 多训练任务管理
- 禁止同时运行多个训练任务（除非使用不同 GPU）
- 如需并行训练，必须指定不同的 CUDA_VISIBLE_DEVICES
- 记录每个训练任务的 GPU 占用情况

## API 服务进程管理
- API 服务使用 uvicorn 启动，设置合理的 worker 数量
- 服务停止时先 graceful shutdown，等待请求处理完成
- 服务异常退出时自动重启（通过 systemd 或 supervisor）
