# 安全规范

## 密钥管理
- 禁止在代码中硬编码密钥、密码、token
- 使用环境变量或密钥管理服务存储敏感信息
- 配置文件中的敏感信息使用占位符：`api_key: ${API_KEY}`

## 输入安全
- API 输入必须校验和清理
- 文件上传必须校验文件类型（魔数检查，非仅扩展名）
- 防止路径遍历攻击：禁止 `../` 在文件路径中
- 图片处理前检查是否为有效图片格式

## 日志安全
- 日志中禁止记录敏感信息：
  - 用户密码
  - API 密钥
  - 个人身份信息（PII）
- 日志文件权限设置为 640（所有者可读写，组可读）

## 模型安全
- 模型文件使用完整性校验（SHA256）
- 模型加载前验证文件来源可信
- 防止模型文件被恶意替换

## 网络安全
- API 服务使用 HTTPS（生产环境）
- 设置合理的超时时间（连接超时、读取超时）
- 限制请求体大小，防止 DoS 攻击

## 容器安全
- Docker 容器以非 root 用户运行
- 最小化镜像，移除不必要的工具
- 定期更新基础镜像，修复安全漏洞
- 使用 .dockerignore 排除敏感文件

## 依赖安全
- 定期检查依赖库的安全漏洞
- 使用固定版本号，避免供应链攻击
- requirements.txt 中使用哈希校验

## 代码安全
- 禁止使用 eval()、exec() 执行动态代码
- 禁止使用 pickle 反序列化不可信数据
- 使用 ast.literal_eval() 替代 eval() 解析配置

## 访问控制
- API 服务支持认证机制（API Key 或 JWT）
- 不同用户设置不同的访问权限
- 记录所有访问日志，便于审计
