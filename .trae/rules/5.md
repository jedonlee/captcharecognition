# 模型保存与加载

## Checkpoint 内容
保存时必须包含完整字典：
- model_state_dict
- optimizer_state_dict
- epoch
- best_val_accuracy
- 完整 config 超参快照（便于独立复现）
- 训练耗时统计
- 随机种子状态（可选，用于精确复现）

## 保存策略
- `checkpoints/best_model.pth` — 验证集最高准确率时覆盖
- `checkpoints/latest_checkpoint.pth` — 每个 epoch 结束更新
- 核心模型和基线模型用不同文件名，避免覆盖
- 保存时使用临时文件 + 原子重命名，防止写入中断导致文件损坏

## 保存时机
- 每个 epoch 结束保存 latest_checkpoint.pth
- val_accuracy 超过历史最佳时保存 best_model.pth
- 训练异常中断时保存 emergency_checkpoint.pth

## 权重加载容错
- 尺寸不匹配的键 → `logger.warning()` 并跳过，不中断
- 缺失的键 → `logger.warning()` 并随机初始化
- 多余的键 → 忽略
- 禁止因形状不符而崩溃

## 加载验证
- 加载后必须验证关键层的权重是否为非零值
- 加载后运行一次前向传播验证输出形状正确
- 记录加载的 checkpoint 文件大小和修改时间

## 本项目约定
- 核心模型 checkpoint：best_model.pth
- 基线模型 checkpoint：baseline_vgg_best.pth
- 消融实验 checkpoint：ablation_{exp_name}_best.pth

## Checkpoint 清理
- 保留最近 3 个 epoch checkpoint（节省磁盘空间）
- best_model.pth 和 latest_checkpoint.pth 永不自动删除
- 清理前必须确认新 checkpoint 保存成功
