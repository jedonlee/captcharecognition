# 性能与监控规范

## 性能基准
- 模型推理延迟：
  - GPU：单图 < 50ms，批量 < 30ms/张
  - CPU：单图 < 200ms，批量 < 150ms/张
- 训练速度：
  - 每个 epoch < 10 分钟（15万样本，V100 GPU）
- API 响应时间：
  - P50 < 50ms，P95 < 100ms，P99 < 200ms

## 性能监控指标
- 训练监控：
  - train_loss / val_loss 曲线
  - val_accuracy 变化趋势
  - 学习率变化曲线
  - 梯度范数分布
  - GPU 利用率和显存占用
- 推理监控：
  - 单次推理延迟
  - 批量推理吞吐量
  - 模型加载时间

## TensorBoard 配置
- 训练指标实时写入 TensorBoard
- 记录标量：loss、accuracy、learning_rate
- 记录直方图：权重分布、梯度分布
- 记录图像：混淆矩阵、样本预测结果

## 告警规则
- 训练异常告警：
  - loss 突然增大（> 前一 epoch 的 2 倍）
  - accuracy 连续 5 个 epoch 无提升
  - 梯度范数异常大（> 10）或为 0
  - GPU 显存占用 > 95%
- 推理异常告警：
  - 单次推理延迟 > 1 秒
  - 批量推理延迟 > 500ms/张
  - 错误率 > 1%

## 性能优化策略
- 模型优化：
  - 使用 AMP 混合精度训练
  - 使用 torch.compile() 编译模型（PyTorch 2.0+）
  - 使用 TensorRT 优化推理（生产环境）
- 数据优化：
  - 使用 num_workers 并行加载
  - 使用 pin_memory 加速数据传输
  - 使用 prefetch 预取数据
- 推理优化：
  - 批量推理代替单图推理
  - 使用 inference_mode() 代替 no_grad()
  - 缓存模型避免重复加载

## 基准测试
- 定期运行基准测试，对比性能变化
- 记录性能基线，用于回归检测
- 性能下降 > 10% 需要调查原因

## 资源利用率目标
- GPU 利用率：> 80%（训练时）
- CPU 利用率：< 90%（避免成为瓶颈）
- 内存利用率：< 85%（避免 swap）
- 磁盘 I/O：< 80%（避免 I/O 瓶颈）
