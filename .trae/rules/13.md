# Docker 与部署规范

## 镜像构建规范
- 使用多阶段构建减小镜像体积
- 基础镜像选择：`nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu22.04`
- 固定基础镜像版本，避免意外更新
- 使用 .dockerignore 排除无关文件

## Dockerfile 规范
```dockerfile
# 必须包含的指令
FROM nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu22.04
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["python", "-m", "uvicorn", "api.app:app", "--host", "0.0.0.0", "--port", "8000"]
```

## 容器运行规范
- 使用 `--gpus all` 启用 GPU 支持
- 限制容器资源：CPU、内存、GPU
- 使用非 root 用户运行容器
- 挂载数据卷而非复制数据

## 部署检查清单
- [ ] 镜像构建成功
- [ ] 容器启动正常
- [ ] 健康检查通过
- [ ] API 接口响应正常
- [ ] GPU 可用且推理正常
- [ ] 日志输出正常
- [ ] 监控指标正常

## 本项目部署命令
```bash
# 构建镜像
docker build -t captcha-api .

# 运行容器
docker run --gpus all -p 8000:8000 -v /data/models:/app/checkpoints captcha-api

# 健康检查
curl http://localhost:8000/health
```

## 环境变量配置
- `CUDA_VISIBLE_DEVICES` — GPU 设备选择
- `MODEL_PATH` — 模型文件路径
- `LOG_LEVEL` — 日志级别
- `WORKERS` — API 工作进程数

## 日志与监控
- 容器日志输出到 stdout/stderr
- 使用 Docker 日志驱动收集日志
- 集成 Prometheus 指标暴露
- 配置容器健康检查

## 回滚策略
- 保留最近 3 个版本的镜像
- 部署失败时自动回滚到上一版本
- 记录每次部署的版本号和变更内容

## CI/CD 集成
- 代码提交自动触发镜像构建
- 测试通过后自动部署到 staging 环境
- 人工确认后部署到 production 环境
- 部署完成后自动运行冒烟测试
