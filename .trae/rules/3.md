# 日志规范

## 统一使用 logging
- 每个文件顶部：
  ```python
  import logging
  logger = logging.getLogger(__name__)
  ```

## 日志级别使用规范
- DEBUG：详细调试信息（如张量形状、中间值）
- INFO：关键流程节点（如训练开始/结束、模型加载、数据集大小）
- WARNING：非致命异常（如配置缺失使用默认值、权重加载不完整）
- ERROR：致命异常（如文件不存在、CUDA OOM、配置校验失败）

## 日志内容要求
- 必须包含上下文信息（如文件名、函数名、关键参数值）
- 错误日志必须包含异常堆栈：`logger.error(f"xxx failed: {e}", exc_info=True)`
- 禁止日志中包含敏感信息（密钥、密码、token）

## 训练日志规范
- 每个 epoch 记录：epoch号、train_loss、val_loss、val_accuracy、learning_rate、耗时
- 每 N 个 batch 记录：当前loss、梯度范数（用于监控训练稳定性）
- 训练结束记录：总耗时、最佳epoch、最佳accuracy

## 禁止
- 使用 print() 输出日志（除 __main__ 的用户交互输出外）
- 日志中使用 f-string 拼接大对象（如完整张量）
- 日志级别滥用（如用 ERROR 记录非异常情况）

## 日志文件管理
- 日志文件按大小轮转（max_bytes=10MB）
- 保留最近 5 个备份文件
- 训练日志和评估日志分开存储
